## 02:30 行业板块数据源修复（用户要求处理）
- **用户反馈**：push2 行业板块在沙盒里 ProxyError，代理是 Clash Verge 规则模式
- **诊断过程**：
  1. `printenv | grep -i proxy` 发现沙盒残留 `127.0.0.1:7897`（Clash 端口）
  2. 脚本入口加 `os.environ.pop` 清除，但 `requests` 仍 RST
  3. 改 `env -u` 在子进程层清除，仍然 RST
  4. curl 直接测也是 `exit 52`（0.1s 服务端 RST）
  5. 改用 `subprocess.check_output(["curl", ...])` 调用 curl，**仍失败**（Python TLS 不通）
  6. 改用 `http://` 协议（之前 curl 测试这个通）→ 502 服务器拒
  7. 扫描 `push2.*.eastmoney.com` 所有子域名 → **`push2delay.eastmoney.com` 返回 200！**
  8. 验证 push2delay 返回完整 10 行行业数据（航空机场 +3.93% 领涨）
- **最终方案**：fetch_real_data() 行业板块请求按 `("push2delay", "push2")` 顺序尝试，第一个成功就 break
- **结果**：6/6 数据源全通，30,838 字节报告，真实行业数据落地
- **教训**：
  - 沙盒里 Python requests 不通但 curl 通的怪异现象，原因未明（可能是 Python TLS 指纹被风控）
  - 子域名扫描是解决"被风控"问题的银弹，东财至少有 push2/push2delay 两个镜像
  - 备用源降级链要尽量长：镜像1 → 镜像2 → akshare → mock
